写这篇文章起源于LeetCode上的一道题 面试题 17.10. 主要元素：

题目描述

数组中占比超过一半的元素称之为主要元素。给你一个整数数组，找出其中的主要元素。若没有，返回-1。请设计时间复杂度为 O(N) 、空间复杂度为 O(1) 的解决方案。

看到一个评论很有意思，把这个算法成为同归于尽算法：

算法分析

假设不同值的元素分属不同阵营，每个元素在战斗时都会和对手同归于尽，那么主要元素阵营在经过随机大乱斗后一定是最后剩下来的元素，因为它比总人数的一半还多。

具体算法如下：

维护两个变量：候选值candidate和次数cnt，遍历数组，cnt = 0时，candidate = nums[i]，candidate == nums[i]时，cnt ++,否则cnt --.最后得到的candidate为可能的主要元素，再遍历一遍数组判断candidate是否为主要元素。

代码实现

int majorityElement(int* nums, int numsSize)
{
    int candidate,cnt = 0;
    for(int i = 0;i < numsSize;i++)
    {
        if(!cnt) candidate = nums[i];
        if(nums[i] == candidate) cnt++;
        else cnt--;
    }
    cnt = 0;
    for(int i = 0;i < numsSize;i++)
        if(candidate == nums[i]) cnt++;
    return cnt > numsSize / 2 ? candidate : -1;
}

拓展

对于伪主要元素（出现次数大于N / k）,维护(k - 1)个candidate和cnt即可，即利用k个人同归于尽法。
